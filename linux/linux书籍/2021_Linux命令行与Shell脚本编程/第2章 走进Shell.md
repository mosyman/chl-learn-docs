# 二、Shell命令的核心格式与界面
Shell脚本的基础是单个Shell命令，掌握命令行的**提示符含义**和**界面特性**，是使用命令的前提。
## 2.1 命令行提示符的含义
输入命令前的固定字符（如`ben@ubuntu: ~$`），是系统的**状态提示**，各部分含义：

| 部分       | 含义                     | 示例       |
|------------|--------------------------|------------|
| `ben`      | 当前登录的**用户名**     | 普通用户   |
| `@`        | 分隔符                   | -          |
| `ubuntu`   | 计算机名（系统安装时设置）| 主机名     |
| `~`        | 当前所在**工作目录**     | `~`代表家目录 |
| `$`/`#`    | 用户权限标识             | 普通用户`$`，超级用户（root）`#` |

**关键要点**：文档中笔误将超级用户提示符写为波浪线`~`，**正确为井号`#`**，可通过提示符快速判断当前用户是否有管理员权限。

## 2.2 命令行界面（CLI）的特性
命令行界面是无图形化的操作环境，仅能通过键盘输入Shell命令完成操作，核心特点：
1. **对比图形界面**：执行速度更快、功能更强大，但需要记忆命令及用法，对初学者有一定门槛；
2. **Ubuntu的CLI环境**：除桌面图形环境外，系统内置6个纯命令行界面，需**输入用户名+密码**登录（密码输入时**不可见**，需确保输入正确）；
3. **类似环境**：Windows的DOS界面、PowerShell，核心逻辑均为“命令输入→系统执行”。

# 三、Linux系统中获取命令帮助的方法
Linux提供了内置的帮助命令，替代Windows的F1键，核心是**`man`命令**，是学习未知命令的核心工具，也是Shell入门的必备技能。
## 3.1 `man`命令的基本用法
**作用**：获取任意系统命令的**在线帮助手册**，手册包含命令的完整用法、选项、描述等；
**语法**：`man + 待查询命令`（例：查`man`自身的帮助：`man man`）；
**退出帮助手册**：按`q`键（查询中按回车键可翻页查看更多内容）。

## 3.2 帮助手册的核心结构
`man`命令输出的手册按固定格式排版，核心板块（以`man man`为例）：
- **NAME**：命令名称+核心作用；
- **SYNOPSIS**：命令的**语法格式**，包含所有可选/必选选项（如`man [-C 文件] [-d] 命令`）；
- **DESCRIPTION**：命令的详细描述，包括使用场景、工作原理；
- **OPTIONS**：所有可选参数的含义（如`-d`表示调试模式）；
- **HISTORY**：命令的开发/维护历史（非核心，可忽略）。

## 3.3 命令不存在的错误处理
若输入的命令系统未识别，会提示`xxx: 未找到命令`，原因是该命令未安装，解决方法：
- Ubuntu系统中，通过**`apt-get`工具**安装（包管理工具，后续章节讲解）；
- 也可通过系统“软件中心”图形化安装。

# 四、第一个Shell脚本：Hello, Bash Shell
所有编程语言的入门案例均为`Hello World`，Shell脚本也不例外，这部分讲解了**Shell脚本从创建到执行的完整流程**，是编写Shell脚本的基础框架。
## 4.1 Shell脚本的创建流程
脚本本质是**存储Shell命令的纯文本文件**，后缀一般为`.sh`（约定俗成，非强制），创建执行的核心流程：
`创建文本文件` → `编写脚本代码` → `赋予可执行权限` → `运行脚本` → `报错修改/运行成功`

## 4.2 实操步骤（以示例2-1.sh为例）
### 步骤1：创建脚本文件
可使用多种工具创建，文档推荐**Gedit文本编辑器**（Ubuntu自带图形化编辑器）：
- 打开：【应用程序】→【文本编辑器】；
- 也可使用命令创建：`touch 2-1.sh`（空文件）、`vim 2-1.sh`（命令行编辑器）。

### 步骤2：编写核心脚本代码
示例2-1.sh的完整代码（带注释），**每行命令单独书写**，核心代码如下：
```bash
# 第一个shell脚本，功能是输出字符串“Hello,Bash Shell”  <-- 注释行
#!/bin/bash                                          <-- 解释器指定行
echo "Hello,Bash Shell"                              <-- 输出字符串
echo                                                 <-- 输出空行
```
**代码逐行解释**：
1. **注释行**：以`#`开头，仅作说明，**不会被Shell执行**，用于标注脚本功能/语句含义，提升可读性；
2. **`#!/bin/bash`**：**核心行，非注释**，以`#!`（称为**shebang**）开头，告知系统执行该脚本需使用**Bash解释器**（`/bin/bash`是Bash的系统路径）；
    - 若省略此行，系统会使用**默认Shell**（大部分Linux默认为Bash，可通过`$SHELL`命令查看默认Shell）；
3. **`echo`命令**：Shell的核心输出命令，作用是将后续的字符串/内容打印到终端；
    - `echo "内容"`：输出指定字符串；
    - 单独`echo`：输出一个空行，用于格式化输出结果。

### 步骤3：保存脚本文件
- 快捷键：`Ctrl+S`，自定义**文件名**（如`2-1.sh`）和**保存路径**（如用户家目录）；
- 命名规范：尽量见名知意（如`hello.sh`），避免纯数字，方便后续查找。

### 步骤4：赋予脚本**可执行权限**
Linux中，**任何文件默认无执行权限**，即使是脚本文件，也需通过`chmod`命令赋予权限，否则执行会提示**权限不足**。
1. **赋权命令**（文档示例）：`chmod u+x 2-1.sh`
    - 含义：`u`代表**文件所有者**（当前用户），`+x`代表**添加可执行权限**；
    - 若需让所有用户都能执行：`chmod a+x 2-1.sh`（`a`代表all，所有用户）。
2. **权限不足的错误提示**：`-bash: ./2-1.sh: 权限不够`，出现此提示即说明未赋权。

### 步骤5：运行Shell脚本
赋权后即可运行，核心有**两种运行方式**，文档重点讲解第一种：
#### 方式1：绝对/相对路径执行（推荐）
**语法**：`./脚本名.sh`（相对路径，代表**当前目录**的脚本），示例：
```bash
./2-1.sh
# 输出结果：
Hello,Bash Shell
（空行）
```
**关键要点**：`./`不可省略，若直接输入`2-1.sh`，系统会在**系统环境变量路径**中查找该脚本，若未找到则提示“无此文件”。

#### 方式2：指定Shell解释器执行
**语法**：`Shell解释器 脚本名.sh`，示例：
```bash
# 使用Bash执行
bash 2-1.sh
# 使用sh执行（sh是Bash的简化版，大部分场景兼容）
sh 2-1.sh
```
**特点**：无需依赖脚本中的`#!/bin/bash`行，直接用指定的Shell解释器解析脚本；但需注意**Shell语法兼容问题**（如ksh、csh的语法与Bash有差异）。

### 步骤6：报错修改
若运行脚本时出现错误（如语法错误、命令错误），需回到编辑器修改代码，保存后重新运行，直至无报错。

## 4.3 Shell脚本的关键格式规范
从入门开始养成规范，避免后续脚本出现莫名错误，核心规范：
1. **注释**：必加！至少在脚本开头标注功能、作者、创建时间，复杂语句后加单行注释；
2. **行格式**：**一条命令占一行**，行尾**无任何符号**（如分号、逗号）；
    - 若需一行写多条命令，用**分号`;`**分隔（例：`echo 1; echo 2`）；
3. **编辑器设置**：
    - Gedit中开启**显示行号**（右下角行列提示区勾选），方便定位错误行；
    - 设置**制表符宽度为4个空格**，替代Tab键，避免不同编辑器中脚本布局错乱；
4. **后缀**：建议使用`.sh`作为后缀，方便识别脚本文件（系统不强制，无后缀也可执行）。

# 五、文档中的笔误/易错点修正
文档中存在几处笔误和易混淆点，**实操时需注意修正**，否则会导致操作错误：
1. 超级用户提示符：文档写为`~`，**正确为`#`**；
2. 命令行界面登录：文档重复描述“输入密码”，核心只需记住**密码输入不可见**，直接输入后按回车即可；
3. `apt-gct`：文档笔误，**正确为`apt-get`**（Ubuntu包管理工具）；
4. 脚本执行路径：文档中`/2-1.sh`是笔误，**正确为`./2-1.sh`**（`.`代表当前目录）。

本章节的所有内容都是Shell脚本的**入门基础**，核心掌握3点即可快速上手：

2. **查帮助**：任何未知命令都用`man 命令`查询，是Linux的“万能词典”；
3. **脚本流程**：`创建→编写（加#!解释器）→赋权（chmod +x）→运行（./脚本.sh）`，这是所有Shell脚本的通用执行框架。
