

# 一、帮助查询命令：man
### 核心功能
查看任意Linux命令的**官方帮助信息**（含语法、选项、使用场景），是Linux入门最基础的“查字典”命令。
### 语法
`man [命令名称]`
### 实例
`[root@test~]# man cd`  # 查看cd命令的完整帮助文档
### 注意
所有命令的详细用法都可通过`man`查询，遇到陌生命令优先使用该命令。

# 二、目录文件查看命令：ls
### 核心功能
列出指定目录下的**文件、子目录**，等效于Windows的`dir`命令，无指定目录时默认查看**当前工作目录**。
### 语法
`ls [选项] [目录或文件]`
### 常用选项（高频必记）
| 选项 | 核心说明 | 易错点 |
|------|----------|--------|
| -a   | 显示**所有**文件/目录，包含**隐藏文件**（Linux中以`.`开头的为隐藏文件） | 会显示`.`（当前目录）和`..`（上级目录） |
| -A   | 显示所有文件/目录，**排除`.`和`..`** | 比-a更实用，避免冗余信息 |
| -l   | 以**长格式**显示详细信息（权限、所有者、大小、修改时间等） | 简写为`ll`（Linux常用别名，等效`ls -l`） |
| -d   | 仅显示**目录本身**的属性，而非目录内的内容 | 常用于查看目录权限，如`ls -ld /root` |
| -t   | 按**修改时间**排序（最新在前） | 结合-l使用：`ls -lt`（按时间看详细信息） |
| -T   | 按时间**倒序**显示（最旧在前） | 可搭配-l：`ls -lT` |
### 高频实例
1. `ls -l /root/`  # 查看/root目录下所有内容的详细信息
2. `ls -lrt /root/`  # 按时间倒序+长格式显示/root目录内容（r=reverse，反向）

# 三、目录切换命令：cd
### 核心功能
切换当前工作目录，是Linux最常用的命令之一。
### 语法
`cd [选项/目录名]`
### 常用选项（无额外参数，均为固定用法）
| 用法 | 核心说明 | 实操场景 |
|------|----------|----------|
| cd [目录名] | 切换到**指定目录**（支持绝对路径/相对路径） | `cd /usr/local`（绝对路径）、`cd test`（相对路径，当前目录下的test） |
| cd / | 切换到Linux**根目录**（所有目录的顶层目录） | 回到系统最顶层 |
| cd .. | 切换到**上级目录**（`cd ../`等效） | 从`/usr/local`切到`/usr` |
| cd ~ | 切换到**当前登录用户的家目录** | root用户家目录为`/root`，普通用户为`/home/用户名` |
| cd | 无参数，**等效cd ~** | 快速回到家目录，最常用 |
### 配套命令
`pwd`：显示当前工作目录的**绝对路径**，切换目录后用`pwd`验证所在位置，如：
```
[root@test ~]# cd /usr/local
[root@test local]# pwd  # 输出：/usr/local
```

# 四、用户创建命令：useradd
### 核心功能
在Linux系统中**创建新的系统用户**，仅root管理员可执行（普通用户无权限）。
### 语法
`useradd [选项] 用户名`
### 常用选项
| 选项 | 核心说明 | 默认值 |
|------|----------|--------|
| -d   | 指定用户的**家目录** | `/home/用户名`（root家目录为`/root`） |
| -g   | 指定用户的**所属主组** | 自动创建与用户名同名的组 |
| -M   | 不自动创建用户的家目录 | 默认会自动创建家目录 |
| -u   | 指定用户的**UID**（用户唯一标识，数字） | 系统自动分配（普通用户UID一般≥1000） |
### 实例
1. 基础创建：`useradd mingongge`
    - 系统自动创建家目录`/home/mingongge`，自动分配UID，创建同名组`mingongge`；
    - 用`tail -1 /etc/passwd`验证（`/etc/passwd`是Linux用户配置文件）。
2. 自定义创建：`useradd mgg -d /root/mgg -u 9999`
    - 指定家目录为`/root/mgg`，UID为9999；
    - 验证：`tail -1 /etc/passwd` 输出`mgg:x:9999:9999::/root/mgg:/bin/bash`。

# 五、密码管理命令：passwd
### 核心功能
**设置/修改Linux用户的密码**，root管理员和普通用户均可执行，权限不同。
### 语法
`passwd [用户名]`
### 不同用户的使用规则
| 执行用户 | 语法/规则 | 实例 |
|----------|-----------|------|
| root管理员 | 可修改**任意用户**密码，无需验证旧密码 | `passwd mgg`（直接修改mgg用户密码） |
| 普通用户 | 仅能修改**自己**的密码，需先验证旧密码 | `passwd`（无参数，直接修改自身密码） |
### 实例说明
1. root修改普通用户密码：
```
[root@test~]# passwd mgg
Changing password for user mgg.
New password:  # 输入新密码（输入时无明文显示）
Retype new password:  # 再次输入新密码
passwd: all authentication tokens updated successfully.  # 修改成功
```
2. 普通用户修改自身密码：
```
[mingongge@test~]$ passwd
Changing password for user mingongge.
(current) UNIX password:  # 先输入旧密码
New password:  # 输入新密码
Retype new password:  # 再次输入
passwd: all authentication tokens updated successfully.
```
### 注意
Linux默认对密码有复杂度要求（如长度≥8位），若输入简单密码会提示`BAD PASSWORD`，但root可强制设置。

# 六、内存查看命令：free
### 核心功能
查看系统**内存使用状态**，包含**物理内存（Mem）、虚拟内存（Swap）、系统缓存**，是运维监控的高频命令。
### 语法
`free [选项]`
### 常用选项
| 选项 | 核心说明 | 实用度 |
|------|----------|--------|
| -b/-k/-m | 分别以**字节/KB/MB**为单位显示 | -m最常用（直观） |
| -h | 以**人类友好格式**显示（自动适配单位：K/M/G） | 推荐使用，如`free -h` |
| -s <秒数> | 持续监控内存，每隔指定秒数刷新一次 | `free -s 5`（每5秒刷新） |
| -t | 显示**内存总和**（Mem+Swap） | 查看系统总内存资源 |
| -V | 显示free命令的版本信息 | 极少用 |
### 实例
1. `free -m`  # 以MB为单位显示内存使用（核心列：total=总内存，used=已用，free=空闲，buff/cache=缓存）
2. `free -t`  # 显示内存总和，默认单位KB

# 七、当前用户查询命令：whoami
### 核心功能
快速显示**当前登录到系统的用户名**，常用于验证当前操作的用户身份（避免权限错误）。
### 语法
`whoami [选项]`
### 常用选项
仅两个辅助选项，无核心功能选项：
- `--help`：查看帮助信息
- `--version`：查看版本信息
### 实例
```
[root@test~]# whoami  # 输出：root（当前为root管理员）
[mingongge@test~]$ whoami  # 输出：mingongge（当前为普通用户）
```

# 八、进程查看命令：ps
### 核心功能
显示系统**当前进程的状态**（进程是Linux中运行的程序实例），是排查进程问题的核心命令。
### 语法
`ps [选项]`
### 常用选项（高频组合必记）
| 选项 | 核心说明 | 常用组合 |
|------|----------|----------|
| -A | 显示**所有进程** | 单独使用：`ps -A` |
| -a | 显示所有用户的进程，含完整路径 | 与-u搭配：`ps -au` |
| -u | 显示进程的**所有者、起始时间** | 搭配a/f：`ps -aux`（最经典组合） |
| -f | 以**全格式**显示进程详细信息 | 搭配e：`ps -ef`（常用） |
| -e | 显示除**系统内核**外的所有进程 | 搭配f：`ps -ef` |
| PID | 查看**指定PID**的进程（PID=进程唯一标识） | `ps -ef | grep 1234`（过滤PID=1234的进程） |
### 经典实例：`ps -ef`
输出结果包含**7个核心列**，是运维必认的字段：
```
UID        PID  PPID  C STIME TTY          TIME CMD
root         1     0  0 09:20 ?        00:00:01 /usr/lib/systemd/systemd
```
| 列名 | 核心含义 |
|------|----------|
| UID  | 运行进程的**用户ID** |
| PID  | 进程的**唯一标识**（杀进程用`kill PID`） |
| PPID | 进程的**父进程ID**（谁创建了这个进程） |
| C    | 进程的**CPU占用率**（百分比） |
| STIME| 进程的**启动时间** |
| TTY  | 启动进程的**终端**（`?`表示后台进程，无终端） |
| TIME | 进程占用的**CPU总时间** |
| CMD  | 进程对应的**执行命令** |
### 补充
`ps -ef | grep [进程名/PID]`：Linux管道过滤，快速查找指定进程（最常用的实操方式）。

# 九、时间管理命令：date
### 核心功能
**显示/修改**Linux系统的**时间和日期**，root可修改，普通用户仅能查看。
### 语法
1. 显示时间：`date [选项] [+时间格式]`（时间格式必须以`+`开头）
2. 修改时间：`date -s [指定时间]`
### 核心用法（分「显示」和「修改」两类）
#### 1. 显示时间（常用格式）
| 格式 | 说明 | 实例 | 输出效果 |
|------|------|------|----------|
| %Y   | 4位完整年份 | `date +%Y` | 2024 |
| %m   | 月份（01-12） | `date +%m` | 10 |
| %d   | 日期（01-31） | `date +%d` | 26 |
| %H   | 小时（00-23） | `date +%H` | 15 |
| %M   | 分钟（00-59） | `date +%M` | 30 |
| %S   | 秒（00-59） | `date +%S` | 59 |
| %T   | 时间（hh:mm:ss） | `date +%T` | 15:30:59 |
| %D   | 日期（mm/dd/yy） | `date +%D` | 10/26/24 |
#### 2. 修改时间（root专属）
| 语法 | 说明 | 实例 |
|------|------|------|
| date -s 年MMDDhhmm | 按「年月份日期小时分钟」修改 | `date -s 20000505`（设为2000年5月5日） |
| date -s "yyyy-mm-dd hh:mm:ss" | 按完整格式修改 | `date -s "2024-10-26 15:30:00"` |
#### 3. 时间偏移查询
查看指定时间的**前/后**时间，无需修改系统时间：
- `date -d '5 day ago'`  # 显示5天前的时间
- `date -d '+5 days'`  # 显示5天后的时间
- `date -d '1 hour ago'`  # 显示1小时前的时间
### 基础实例
```
[root@test~]# date  # 直接显示系统当前时间：Sat May 5 15:35:23 CST 2018
[root@test~]# date '+Today is:%D,now is:%T'  # 自定义格式：Today is:05/05/18,now is:15:40:03
```

# 十、当前目录查询命令：pwd
### 核心功能
显示当前工作目录的**绝对路径**，解决“不知道自己在哪”的问题，与cd命令配套使用。
### 语法
`pwd [选项]`
### 常用选项
仅`--help`：查看pwd命令的帮助信息，无其他核心选项。
### 实例
```
[root@test local]# pwd  # 输出：/usr/local（当前在/usr/local目录）
[root@test ~]# pwd  # 输出：/root（当前在root家目录）
```

# 十一、系统关机/重启命令：shutdown
### 核心功能
对Linux系统执行**关机、重启**操作，可延迟执行，比`poweroff`/`reboot`更安全（会先保存进程、关闭服务）。
### 语法
`shutdown [选项] [时间]`（时间省略则默认延迟1分钟）
### 常用选项
| 选项 | 核心说明 | 实操用法 |
|------|----------|----------|
| -h   | 执行**关机**操作 | `shutdown -h now`（立即关机）、`shutdown -h 10`（10分钟后关机） |
| -r   | 执行**重启**操作 | `shutdown -r now`（立即重启）、`shutdown -r 5`（5分钟后重启） |
| -t <秒数> | 推迟指定秒数后执行操作 | `shutdown -h -t 30 now`（30秒后立即关机） |
| -f   | 重启时**不执行fsck**（文件系统检查） | 加快重启速度，无特殊需求不使用 |
### 核心实例
`[root@test~]# shutdown -h now`  # 系统立即关机，执行后远程连接会直接断开
### 补充
- 取消未执行的shutdown：`shutdown -c`（如执行`shutdown -h 10`后，可通过此命令取消）；
- 仅root管理员可执行shutdown命令。

# 整体学习要点
1. **权限区分**：所有**用户管理、系统修改、关机重启**的命令（useradd、passwd、date、shutdown）仅**root管理员**可执行，普通用户无权限；
2. **选项组合**：Linux命令的核心是**选项组合**（如`ls -lrt`、`ps -ef`、`free -mh`），单选项无意义，需按场景搭配；
3. **配套使用**：cd与pwd、useradd与passwd、ps与grep是固定配套命令，实操中需结合使用；
4. **文件路径**：Linux中**绝对路径**以`/`开头（如`/usr/local`），**相对路径**以当前目录为基准（如`test`），切换目录时优先使用绝对路径避免错误。
