## 一、Linux 目录结构总览

Linux 采用**树形目录结构**，所有目录都挂载在根目录 `/` 之下，并且按类别划分，有清晰的层级关系。

- **根目录 `/`**：整个文件系统的起点，所有目录和文件都从这里开始。
- **目录划分原则**：按功能和用途分类，比如存放命令、配置、日志、用户数据等，方便管理和维护。

## 二、根目录下的核心目录及作用（2.1.1）

| 目录 | 作用说明 |
|------|----------|
| `/bin` | 存放**所有二进制命令**（用户命令），如 `ls`、`cp`、`rm` 等，是系统启动和运行必备的命令。 |
| `/boot` | 存放内核及引导系统程序，如 `vmlinuz`、`initramfs` 等，系统启动时会加载这里的文件。 |
| `/dev` | 存放设备文件，如硬盘 `/dev/sda`、光驱 `/dev/cdrom` 等，Linux 把硬件都抽象成文件。 |
| `/etc` | 系统配置文件目录，是**最重要的目录之一**，几乎所有服务和系统的配置都在这里。 |
| `/home` | 普通用户的家目录，如用户 `alice` 的家目录是 `/home/alice`，`root` 用户的家目录是 `/root`。 |
| `/lib` | 32位库文件存放目录，是系统运行所需的共享库。 |
| `/lib64` | 64位库文件存放目录，64位系统的共享库。 |
| `/media` | 媒体文件存放目录，用于挂载可移动设备（如U盘、光盘）。 |
| `/mnt` | 临时挂载存储设备（如U盘、硬盘）的目录，管理员手动挂载时常用。 |
| `/opt` | 自定义软件安装存放目录，一些第三方软件会安装在这里。 |
| `/proc` | 虚拟文件系统，存放系统内核和进程信息，是内存中的数据映射。 |
| `/root` | 管理员 `root` 用户的家目录，和普通用户的 `/home` 分开。 |
| `/run` | 系统运行时产生的临时文件存放目录，如 PID 文件、套接字等。 |
| `/sbin` | 存放**系统管理员命令**（只有 root 能执行的命令），如 `reboot`、`shutdown`、`ifconfig` 等。 |
| `/srv` | 服务启动后需要访问的数据目录，如网站数据、FTP 数据等。 |
| `/sys` | 系统使用目录，存放与硬件相关的信息，是内核导出的设备和驱动信息。 |
| `/tmp` | 临时文件目录，所有用户都可读写，重启后文件会被清空。 |
| `/usr` | 系统命令和帮助文件目录，存放用户级的应用程序和文件，如 `/usr/bin`、`/usr/lib`。 |
| `/var` | 存放内容经常变动的文件目录，如日志、缓存、数据库文件等。 |

## 三、根下常见目录详解（2.1.2 & 2.2）

### 1. `/bin` 目录
- 存放**用户二进制命令**，是系统启动和日常操作必备的命令集合。
- 示例：`ls`、`cp`、`mv`、`rm`、`cat` 等。
- 书中提到：`/bin` 目录下有 731 个文件，代表系统安装了 731 个二进制命令。

### 2. `/etc` 目录（重点）
这是**系统配置文件目录**，是运维中最常操作的目录之一。

| 文件 | 作用 |
|------|------|
| `/etc/resolv.conf` | 设置本地客户端 DNS 的文件，决定系统如何解析域名。 |
| `/etc/fstab` | 记录开机需要挂载的文件系统，系统启动时会自动挂载这里列出的设备。 |
| `/etc/init.d` | 存储系统或服务器以 System V 模式启动的脚本文件（旧版服务管理）。 |
| `/etc/inittab` | 设定系统启动级别的配置文件（在 systemd 为主流的系统中已较少使用）。 |
| `/etc/profile` | 系统全局环境变量配置文件，所有用户登录时都会加载。 |
| `/etc/rc.local` | 存储开机后自动启动程序的命令，可自定义开机自启项。 |

### 3. `/var` 目录（重点）
用于存放**内容经常变动的文件**，如日志、缓存、数据库文件等。

| 文件/目录 | 作用 |
|-----------|------|
| `/var/log/dmesg` | 系统核心启动日志文件，记录内核启动时的硬件检测和初始化信息。 |
| `/var/log/messages` | 系统级日志文件，记录大部分系统服务的运行信息（部分系统已用 `journald` 替代）。 |
| `/var/log/mailog` | 邮件系统日志文件，记录邮件服务的收发信息。 |
| `/var/log/wtmp` | 记录用户登录信息的文件，可通过 `last` 命令查看。 |
| `/var/log/` | 应用服务的日志通常存放在这里，如 `/var/log/httpd` 用于存储 httpd 服务的日志。 |

---

## 四、核心要点总结

1.  **树形结构**：所有目录都从 `/` 开始，层级清晰，便于管理。
2.  **目录分工明确**：命令、配置、日志、用户数据等都有专门的目录，符合“单一职责”原则。
3.  **重点目录**：
    - `/etc`：系统配置的心脏，运维高频操作区。
    - `/var`：日志和动态数据的仓库，排查问题时必看。
    - `/bin`、`/sbin`：命令集合，区分用户命令和管理员命令。
4.  **设备抽象**：`/dev` 目录把硬件设备抽象成文件，体现了“一切皆文件”的设计哲学。
