
## `tee` 命令的官方手册

### 一、tee 命令核心功能（大白话解释）
`tee` 的核心作用是**“分流”** —— 它会读取来自“标准输入”（比如终端输入、管道传递的内容）的信息，同时做两件事：
1. 把内容输出到**标准输出**（默认是终端屏幕）；
2. 把内容写入到指定的**文件**中。

可以把它想象成“水管三通阀”：一根进水管（标准输入），分出两根出水管（屏幕 + 文件），水流（数据）会同时流向两个方向。

### 二、核心参数（OPTIONS）详细解释
手册里的参数可以分为“基础常用”和“高级特殊”两类，我会逐个拆解：

| 参数                | 中文解释                                                                 | 核心用途                                                                 |
|---------------------|--------------------------------------------------------------------------|--------------------------------------------------------------------------|
| `-a, --append`      | 追加模式（append），而非覆盖                                             | 默认情况下 `tee` 会清空文件再写入，加 `-a` 会把内容追加到文件末尾（类似 `>>`） |
| `-i, --ignore-interrupts` | 忽略中断信号（比如按 `Ctrl+C`）| 即使手动按 `Ctrl+C` 中断命令，`tee` 也会先完成数据写入，避免文件内容不完整       |
| `-p`                | 针对管道（pipe）优化的模式                                               | 配合管道使用时，让 `tee` 对管道错误的处理更合理（默认用 `warn-nopipe` 模式）    |
| `--output-error[=MODE]` | 设置写入出错时的行为（MODE 是具体规则）| 控制 `tee` 遇到“写文件/写管道失败”时，是报错、退出还是忽略                   |
| `--help`            | 显示帮助信息并退出                                                       | 快速查看参数说明，不用翻手册                                               |
| `--version`         | 显示版本信息并退出                                                       | 查看当前 `tee` 版本（通常是 GNU coreutils 套件的一部分）                    |

#### 重点：--output-error 的 MODE 规则（最易混淆）
MODE 决定了 `tee` 写入失败时的行为，手册里的 4 种 MODE 可以分成两组理解：
1. **warn 组（只报错，不退出）**
    - `warn`：无论写入“文件”还是“管道”失败，都只打印错误信息，继续运行；
    - `warn-nopipe`：只有写入“非管道”（比如文件）失败时报错，写入管道失败不报错；
2. **exit 组（报错并退出）**
    - `exit`：无论写入“文件”还是“管道”失败，直接退出 `tee` 命令；
    - `exit-nopipe`：只有写入“非管道”（比如文件）失败时退出，写入管道失败不退出。

**默认行为（没指定 --output-error 时）**：
- 写入管道失败 → 立刻退出；
- 写入文件失败 → 只打印错误，不退出。

### 三、实用示例（结合场景理解）
光看参数太抽象，结合实例才能真正掌握：

#### 示例 1：基础用法（同时输出到屏幕和文件）
```bash
# 输入一行文字，同时输出到屏幕 + 写入 test.txt
echo "hello tee" | tee test.txt

# 执行结果：
# 屏幕上会显示：hello tee
# 同时生成 test.txt 文件，内容也是 hello tee
```

#### 示例 2：追加模式（-a）
```bash
# 先写入一行，再追加一行（避免覆盖）
echo "第一行" | tee test.txt
echo "第二行" | tee -a test.txt

# test.txt 内容：
# 第一行
# 第二行
```

#### 示例 3：写入多个文件
```bash
# 同时写入 file1.txt 和 file2.txt，屏幕也输出
echo "多文件测试" | tee file1.txt file2.txt
```

#### 示例 4：忽略中断（-i）
```bash
# 模拟长时间写入，按 Ctrl+C 也不会中断
cat large_file.txt | tee -i output.txt
# 即使按 Ctrl+C，tee 也会先完成写入，不会让 output.txt 内容残缺
```

#### 示例 5：结合管道和 --output-error（高级）
```bash
# 强制让管道失败（比如用不存在的命令），测试 exit 模式
echo "测试错误处理" | tee --output-error=exit /dev/full non_exist_pipe
# /dev/full 是系统特殊文件（写入会提示磁盘满），此时 tee 会直接退出并报错
```

2. **报错反馈**：如果发现 `tee` 的 bug，可以反馈到 `bug-coreutils@gnu.org`（一般用户用不到）。
3. **完整文档**：可以通过 `info '(coreutils) tee invocation'` 查看本地完整手册，或访问 GNU 官网。

### 总结
1. `tee` 的核心是**“一分二”**：把标准输入的内容同时输出到屏幕和文件（支持多文件）；
2. 最常用参数是 `-a`（追加）和 `-i`（忽略中断），`--output-error` 主要用于脚本容错；
3. 默认行为：写入文件失败只报错不退出，写入管道失败直接退出（-p 模式默认用 warn-nopipe）。

这个命令在脚本中特别实用，比如执行命令时既要在屏幕看输出，又要把日志保存到文件，不用分开执行两次命令。


## `tee` 命令的用法
`tee` 是 Linux/Unix 系统中非常实用的命令，核心作用是**读取标准输入并同时输出到文件和标准输出**（通俗说就是“一边显示在屏幕上，一边保存到文件里”）。我会从核心语法、常用选项、实操示例、进阶技巧四个维度讲透，让你彻底掌握。

### 一、核心定义与语法
#### 1. 基本作用
`tee` 命令的名字来源于“T型管道”——就像水管分岔，把一份输入分成两份：一份输出到**终端（stdout）**，一份写入**文件**。
#### 2. 基本语法
```bash
command | tee [选项] 文件名1 文件名2 ...
```
- `command`：产生输出的命令（如 `ls`/`cat`/`echo`）；
- `|`：管道符，将 `command` 的输出传给 `tee`；
- `[选项]`：可选参数（如下文的 `-a`/`-i`）；
- `文件名`：要写入的文件（可指定多个，同时写入多文件）。

### 二、核心选项（必记）
| 选项 | 作用 | 示例 |
|------|------|------|
| `-a`/`--append` | 追加模式（默认是覆盖文件） | `ls | tee -a file.txt` |
| `-i`/`--ignore-interrupts` | 忽略中断信号（如 Ctrl+C） | `long_command | tee -i log.txt` |
| `--help` | 查看帮助 | `tee --help` |
| `--version` | 查看版本 | `tee --version` |

### 三、实操示例（最常用场景）
#### 1. 基础用法：同时输出到屏幕+文件（覆盖模式）
```bash
# 执行 ls 命令，结果既显示在屏幕，又保存到 file.txt（覆盖原有内容）
ls -l | tee file.txt

# 验证：查看文件内容
cat file.txt
```
- 屏幕上会显示 `ls -l` 的结果；
- `file.txt` 中也会保存相同的结果；
- 如果 `file.txt` 已存在，**原有内容会被覆盖**。

#### 2. 追加模式（-a）：不覆盖原有文件
```bash
# 追加 ls 结果到 file.txt，保留原有内容
ls -l | tee -a file.txt
```
- 这是生产环境最常用的模式（避免误删文件内容）。

#### 3. 同时写入多个文件
```bash
# 将 echo 的输出同时写入 file1.txt、file2.txt，且显示在屏幕
echo "hello tee" | tee file1.txt file2.txt

# 验证
cat file1.txt  # 输出 hello tee
cat file2.txt  # 输出 hello tee
```

#### 4. 结合 sudo 写入权限文件
场景：普通用户执行命令，但需要将结果写入 root 权限的文件（如 `/etc/docker/daemon.json`）。
```bash
# 错误写法（tee 无权限，会报错）
echo '{"registry-mirrors":["https://mirror.aliyuncs.com"]}' | tee /etc/docker/daemon.json

# 正确写法（给 tee 加 sudo）
echo '{"registry-mirrors":["https://mirror.aliyuncs.com"]}' | sudo tee /etc/docker/daemon.json
```
- 原理：`echo` 由普通用户执行，`tee` 由 root 执行，解决文件写入权限问题（这也是你之前配置 Docker 加速器时用到的写法）。

#### 5. 忽略中断信号（-i）：保证命令执行完成
场景：执行耗时命令（如 `docker pull`），即使按 Ctrl+C，也能保证日志写入文件。
```bash
# 拉取镜像，日志同时存到 pull.log，且忽略 Ctrl+C 中断
docker pull mysql:8.0.45-debian | tee -i pull.log
```
- 即使中途按 Ctrl+C，`pull.log` 仍会保存已执行部分的日志。

#### 6. 配合管道继续处理输出
`tee` 的输出还能通过管道传给下一个命令，实现“保存+二次处理”：
```bash
# 1. 保存 ls 结果到 file.txt；2. 统计文件行数
ls -l | tee file.txt | wc -l
```
- 屏幕会显示 `ls -l` 的行数；
- `file.txt` 保存完整的 `ls -l` 结果。

### 四、进阶技巧（实用场景）
#### 1. 记录命令执行日志（排查问题）
```bash
# 执行一系列命令，全程记录日志到 operation.log
{
  echo "开始执行：$(date)"
  docker pull mysql:8.0.45-debian
  docker images | grep mysql
  echo "执行结束：$(date)"
} | tee -a operation.log
```
- `{ ... }` 把多个命令的输出合并，通过 `tee` 统一记录；
- `-a` 保证日志追加，不会覆盖。

#### 2. 替换文件内容（保留原格式）
```bash
# 替换文件中 "old" 为 "new"，同时备份原文件
cat test.txt | sed 's/old/new/g' | tee test.txt.bak | tee test.txt
```
- 先通过 `sed` 替换内容；
- 先写入 `test.txt.bak`（备份），再写入 `test.txt`（覆盖原文件）。

#### 3. 实时监控命令输出并保存
```bash
# 实时查看系统负载，同时保存到 load.log
top -b -d 1 | tee -a load.log
```
- `-b`：`top` 批处理模式（非交互）；
- `-d 1`：每 1 秒刷新一次；
- 屏幕实时显示负载，`load.log` 持续追加记录。

### 五、常见坑点与注意事项
1. **覆盖 vs 追加**：默认是覆盖模式，生产环境务必加 `-a`，避免误删文件；
2. **权限问题**：写入系统目录（如 `/etc`）需给 `tee` 加 `sudo`，而非给前面的命令加；
3. **空文件问题**：如果管道前的命令无输出（如 `echo ""`），`tee` 会创建空文件；
4. **多个文件写入**：指定多个文件时，`tee` 会给每个文件写入相同内容。

### 总结
1. **核心功能**：`tee` = 「屏幕输出 + 文件写入」二合一，解决“既要看到结果，又要保存日志”的需求；
2. **高频选项**：`-a`（追加）、`-i`（忽略中断）、配合 `sudo`（权限写入）；
3. **核心场景**：配置文件写入、命令日志记录、多文件备份、权限文件修改。
