
研究 Elasticsearch/ELK 要走「**原理 → 实战 → 源码**」的递进路线，优先啃**官方文档**与**源码**，再辅以权威资料。下面给你一套可直接执行的学习路径 + 全量资料清单（含官方入口）。

## 一、整体学习路线（分三阶段）

### 阶段 1：基础与原理（1–2 周）
目标：搞懂 ELK 是什么、怎么工作、核心概念与架构。
1. **ELK 整体架构与组件分工**
    - ELK = Elasticsearch + Logstash + Kibana（现称 Elastic Stack，含 Beats）
    - 数据流：**Beats/Logstash 采集 → ES 存储/检索 → Kibana 可视化**
2. **Elasticsearch 核心原理（重中之重）**
    - 基于 **Apache Lucene** 构建，分布式、RESTful、近实时搜索/分析
    - 核心概念：**索引、文档、分片/副本、倒排索引、集群节点（Master/Data/Coordinating）**
    - 写入/查询流程、分布式一致性、高可用机制
3. **Logstash 原理**
    - 数据 pipeline：**Input → Filter → Output**
    - 插件机制、性能与资源特点（重 JVM，轻量场景用 Beats）
4. **Kibana 原理**
    - 可视化、Dashboard、Dev Tools、与 ES 交互方式
5. **动手搭建**
    - Docker 一键起 ES + Kibana（推荐）
    - 用 Kibana Dev Tools 做 CRUD、查询、聚合练习

### 阶段 2：实战与进阶（2–3 周）
目标：会用、会调优、会排查。
1. **ES 核心 API 与 DSL**
    - 索引管理、文档 CRUD、全文检索、过滤、聚合（Aggregation）
    - Mapping、分词器（IK 中文）、索引模板、ILM 生命周期
2. **ELK 完整链路实战**
    - Filebeat 采集日志 → Logstash 清洗 → ES 存储 → Kibana 可视化
    - 多节点集群部署、安全（x-pack）、监控
3. **性能与调优**
    - 分片规划、写入/查询优化、JVM 与堆内存、磁盘/网络
4. **问题排查**
    - 集群健康、慢查询、GC、分片未分配、数据丢失

### 阶段 3：源码深度（2–4 周+）
目标：读懂核心流程、模块、关键类与算法。
1. **源码环境搭建**
    - 拉取 ES 官方源码、编译、本地调试
2. **核心模块与流程**
    - 集群发现与选举、节点通信、索引写入、查询执行、Lucene 交互
    - 倒排索引构建、FST、评分机制、分片路由
3. **关键类与入口**
    - `Node`、`IndicesService`、`IndexService`、`SearchService`、`TransportService`
    - REST 入口、请求分发、任务执行
4. **扩展与插件**
    - 插件机制、自定义分析器、脚本、REST 端点

## 四、源码研究方法（关键步骤）
1. **拉取源码**
   ```bash
   git clone https://github.com/elastic/elasticsearch.git
   cd elasticsearch
   ```
2. **编译与运行**
    - 用 Gradle 编译：`./gradlew assemble`
    - 本地运行单节点：`./gradlew run`
3. **目录结构概览**
   ```
   elasticsearch/
   ├── server/          # 核心服务（集群、索引、查询、节点通信）
   ├── client/          # 客户端 API
   ├── common/          # 工具类、配置、序列化
   ├── modules/         # 插件式模块（分析、脚本、SQL 等）
   ├── libs/            # 依赖库（Lucene、网络、缓存）
   ├── x-pack/          # 扩展（安全、监控、ML）
   └── distribution/    # 打包部署
   ```

4. **从入口开始读**
    - 主入口：`org.elasticsearch.bootstrap.Elasticsearch`
    - 节点启动：`Node`、`NodeEnvironment`、`ClusterService`
    - REST 请求：`RestController`、`RestHandler`
5. **核心流程断点调试**
    - 写入：`IndexRequest` → `TransportIndexAction` → `IndexService`
    - 查询：`SearchRequest` → `TransportSearchAction` → `SearchService` → Lucene
6. **重点模块**
    - 集群：`ClusterService`、`Discovery`、`MasterService`
    - 索引：`IndexService`、`ShardId`、`Engine`（Lucene 交互）
    - 查询：`SearchService`、`QueryBuilder`、`Collector`、评分机制
    - 网络：`TransportService`、`TcpTransport`

## 五、学习建议
1. **先文档后源码**：原理与 API 先吃透官方文档，再读源码才不盲目。
2. **边学边练**：每学一个概念就用 Kibana Dev Tools 验证。
3. **源码抓主线**：先看懂「写入/查询/集群」三大流程，再深入细节。
4. **善用调试**：本地起源码，断点跟踪请求流转，比纯看代码高效。
5. **从问题驱动**：遇到性能/故障问题，去源码找对应逻辑。
