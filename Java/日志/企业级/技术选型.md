

GitHub 上有大量成熟、可直接部署的**企业级开源日志收集与展示系统**，覆盖从轻量到全链路、从单节点到分布式的各种场景。下面按主流方案分类，给出最常用、最稳定的开源项目。

---

### 一、主流全链路日志平台（收集+存储+查询+可视化+告警）
#### 1. ELK Stack（Elasticsearch + Logstash + Kibana）
- **GitHub**：
    - Elasticsearch：https://github.com/elastic/elasticsearch
    - Logstash：https://github.com/elastic/logstash
    - Kibana：https://github.com/elastic/kibana
- **定位**：最经典、生态最完善的企业级日志方案，支持海量日志、全文检索、复杂分析与可视化。
- **核心能力**：
    - Logstash/Filebeat 收集各类日志（文件、TCP、UDP、容器、云服务）
    - Elasticsearch 分布式存储与全文索引
    - Kibana 强大的仪表盘、图表、告警与日志检索
- **适用场景**：中大型企业、海量日志、复杂业务分析、安全审计。

#### 2. Graylog
- **GitHub**：https://github.com/Graylog2/graylog2-server
- **定位**：轻量版 ELK，开箱即用，更简单的部署与运维。
- **核心组件**：Graylog Server + Elasticsearch + MongoDB
- **优势**：
    - 内置 Web UI，无需额外配置 Kibana
    - 自带流（Stream）、告警、用户权限、日志归档
    - 比 ELK 更轻量，适合中小团队快速落地
- **适用场景**：中小企业、快速搭建、轻量运维。

#### 3. Loki + Promtail + Grafana
- **GitHub**：
    - Loki：https://github.com/grafana/loki
    - Promtail：https://github.com/grafana/promtail
    - Grafana：https://github.com/grafana/grafana
- **定位**：云原生、低成本、Prometheus 风格的日志方案。
- **核心特点**：
    - 不索引日志全文，只索引标签（如 `app=payment`），存储成本极低
    - 与 Prometheus/Grafana 生态无缝集成，统一监控与日志面板
    - 水平扩展、多租户、适合 Kubernetes 环境
- **适用场景**：云原生、K8s、成本敏感、监控+日志一体化。

#### 4. SigNoz
- **GitHub**：https://github.com/SigNoz/signoz
- **定位**：新一代开源可观测平台，统一日志、指标、链路追踪。
- **核心能力**：
    - 基于 ClickHouse 存储，查询极快、成本低
    - 内置日志检索、仪表盘、告警、分布式追踪
    - 支持 OpenTelemetry，一键接入各类服务
- **适用场景**：微服务、云原生、需要全链路可观测的企业。

### 二、轻量/专用日志工具（适合快速部署或特定场景）
#### 1. Seq
- **GitHub**：https://github.com/datalust/seq
- **定位**：结构化日志优先，适合 .NET/Java 等结构化日志场景。
- **特点**：
    - 支持 SQL 风格查询日志
    - 内置仪表盘、告警、日志保留策略
    - 部署简单，单节点即可支撑中小规模
- **适用场景**：结构化日志、中小团队、快速查询。

#### 2. OpenSearch + OpenSearch Dashboards（Elasticsearch 分支）
- **GitHub**：https://github.com/opensearch-project
- **定位**：开源、无商业限制的 Elasticsearch 替代方案。
- **特点**：
    - 与 ELK 兼容，可无缝迁移
    - 完全开源，无许可证风险
    - 内置安全、告警、机器学习插件
- **适用场景**：企业级、需要规避 Elastic 商业条款、大规模日志。

#### 3. Fluentd + Elasticsearch + Kibana
- **GitHub**：https://github.com/fluent/fluentd
- **定位**：云原生日志收集器，常与 ELK 搭配。
- **优势**：
    - 插件极丰富，支持几乎所有日志源与输出
    - 轻量、资源占用低，适合容器/边缘环境
- **适用场景**：K8s、容器化、多源日志统一收集。

---

### 三、日志收集客户端（Agent，用于采集日志）
这些是日志系统的“采集端”，通常配合上面的存储/展示系统使用：
- **Filebeat**（Elastic）：https://github.com/elastic/beats/tree/main/filebeat
    - 轻量、高效，专门收集文件日志，适合 ELK/OpenSearch。
- **Promtail**（Grafana）：https://github.com/grafana/promtail
    - 专为 Loki 设计，支持标签、K8s 服务发现。
- **Fluent Bit**：https://github.com/fluent/fluent-bit
    - 超轻量、高性能，适合边缘/容器，常对接 Loki/Elasticsearch。
- **syslog-ng**：https://github.com/syslog-ng/syslog-ng
    - 企业级 syslog 收集/转发，支持复杂过滤与路由。

### 四、方案对比（快速选型）
| 方案 | 存储 | 部署复杂度 | 成本 | 适合场景 |
|------|------|------------|------|----------|
| ELK | Elasticsearch | 中高 | 中 | 海量日志、全文检索、复杂分析 |
| Graylog | Elasticsearch | 中 | 中 | 中小团队、快速搭建、轻量运维 |
| Loki | 自研索引 | 低 | 极低 | 云原生、K8s、监控+日志一体化 |
| SigNoz | ClickHouse | 中 | 低 | 微服务、全链路可观测 |
| OpenSearch | OpenSearch | 中高 | 中 | 规避 Elastic 商业限制、大规模 |

---

### 五、快速上手建议
1. **中小团队/快速验证**：优先选 **Graylog** 或 **Loki+Grafana**，部署简单、见效快。
2. **云原生/K8s**：优先 **Loki+Promtail+Grafana** 或 **SigNoz**，原生支持容器与服务发现。
3. **海量日志/复杂分析**：选 **ELK** 或 **OpenSearch**，生态成熟、功能最强。
4. **成本敏感**：选 **Loki** 或 **SigNoz**，存储与计算成本远低于 ELK。
