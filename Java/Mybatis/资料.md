
你想要系统地研究 MyBatis 的原理和源码，并且需要一份全面的资料清单（尤其是官方资料），我会帮你梳理出一套清晰的学习路径，并整理好所有核心的学习资源。

### 一、研究 MyBatis 原理与源码的学习路径
#### 1. 基础准备：先掌握使用，再深究原理
在看源码前，必须先熟练掌握 MyBatis 的核心用法，否则看源码会毫无头绪：
- 掌握核心配置：`mybatis-config.xml`（环境配置、别名、插件、映射器）、`Mapper.xml`（CRUD 标签、参数传递、结果映射、动态 SQL）。
- 掌握核心 API：`SqlSessionFactory`、`SqlSession`、`Mapper 接口`、`SqlSessionFactoryBuilder` 的使用。
- 掌握核心特性：参数解析、结果集映射、动态 SQL、插件机制、一级/二级缓存。

#### 2. 源码阅读前的准备工作
- **环境搭建**：
    1. 从 GitHub 克隆 MyBatis 源码：`git clone https://github.com/mybatis/mybatis-3.git `。
    2. 建议选择**稳定版本**（如 3.5.x 系列），避免最新开发版的不稳定。
    3. 导入 IDE（IDEA 最佳），确保 Maven/Gradle 依赖加载完成，能正常编译。
- **核心依赖了解**：MyBatis 核心依赖只有 `slf4j-api`（日志），无其他强依赖，源码阅读成本低。

#### 3. 源码阅读的核心顺序（从入口到核心）
按「执行流程」拆解源码，避免漫无目的地看：
##### （1）入口：SqlSession 的创建过程
- 核心类：`SqlSessionFactoryBuilder` → `XMLConfigBuilder`（解析核心配置文件） → `Configuration`（MyBatis 全局配置容器，核心中的核心）。
- 重点看：`XMLConfigBuilder.parse()` 方法如何解析 `mybatis-config.xml`，并将所有配置存入 `Configuration` 对象。

##### （2）核心：Mapper 接口的代理与 SQL 执行
- 核心类：`MapperProxyFactory`（动态代理创建 Mapper 接口实例）、`MapperMethod`（封装 Mapper 接口的方法和 SQL 信息）、`Executor`（执行器，SQL 执行的核心）。
- 重点看：
    - `MapperRegistry.getMapper()` 如何通过 JDK 动态代理生成 Mapper 代理对象。
    - `Executor` 的子类（`SimpleExecutor`/`ReuseExecutor`/`BatchExecutor`）如何执行 SQL，以及一级缓存的实现。

##### （3）关键：SQL 解析与参数处理
- 核心类：`SqlSource`（动态 SQL 解析后的 SQL 源）、`ParameterHandler`（参数绑定）、`StatementHandler`（Statement 创建与执行）。
- 重点看：动态 SQL（if/where/foreach）的解析过程（`XMLScriptBuilder`）。

##### （4）扩展：结果集映射与缓存
- 结果集映射：`ResultSetHandler` → `DefaultResultSetHandler.handleResultSets()` 如何将数据库结果映射为 Java 对象。
- 缓存：`PerpetualCache`（基础缓存）、`CachingExecutor`（二级缓存执行器），理解一级缓存（SqlSession 级别）和二级缓存（Mapper 级别）的实现。

##### （5）进阶：插件机制
- 核心类：`Interceptor`（插件接口）、`Plugin`（动态代理封装插件）、`InterceptorChain`（插件链）。
- 重点看：MyBatis 如何通过动态代理拦截 `Executor`/`StatementHandler` 等核心对象的方法。

### 三、高效研究源码的小技巧
1. **先画执行流程图**：根据官方文档和基础用法，先手绘 MyBatis 执行 SQL 的核心流程（如：配置解析 → 创建 SqlSession → 获取 Mapper 代理 → 执行 SQL → 结果映射），再对照源码填充细节。
2. **从测试用例入手**：源码中的测试用例（如 `SqlSessionTest`、`MapperProxyTest`）是最小的可运行单元，通过调试测试用例，能快速定位核心逻辑。
3. **聚焦核心类，忽略边缘代码**：优先看 `org.apache.ibatis.session`、`org.apache.ibatis.executor`、`org.apache.ibatis.binding` 核心包，暂时忽略日志、异常处理等边缘代码。
4. **结合问题研究**：带着具体问题看源码（如「MyBatis 如何实现 Mapper 接口无实现类却能执行」），比泛泛而看效率高。

### 总结
1. 研究 MyBatis 源码的核心逻辑是「先会用 → 搭环境 → 按执行流程拆核心类 → 结合调试深挖」，避免直接上手啃源码。
2. 官方资料（文档+源码+测试用例）是最权威的学习依据，优先吃透，补充资料仅作为辅助。
3. 核心突破点：抓住 `Configuration`（全局配置）、`Executor`（执行器）、`MapperProxy`（动态代理）、`StatementHandler`（SQL 执行）这几个核心类，就能掌握 MyBatis 的核心原理。
