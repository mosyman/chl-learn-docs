### 一、核心概念：三种服务器配置类型（对比理解）
首先，先通过表格清晰区分 GoLand 里的三种核心服务器配置，这是理解的基础：

| 配置类型       | 核心特点                                                                 | 适用场景                                   |
|----------------|--------------------------------------------------------------------------|--------------------------------------------|
| 就地服务器     | 直接从**本地 Web 服务器的文档根目录**打开项目，无文件上传/下载/同步       | 本地服务器文档根目录就是项目目录（无需同步） |
| 本地服务器     | 本地 GoLand 项目开发 → 手动/自动上传文件到**本地（或挂载）Web 服务器**    | 项目目录和本地服务器文档根目录分离         |
| 远程服务器     | 服务器在另一台主机，通过 FTP/SFTP/FTPS/WebDAV 协议访问/传输文件           | 代码需要部署到远程服务器（如测试/生产机）   |

#### 关键区别（新手必看）：
- “就地服务器” = 项目文件夹 ≡ 服务器文档根目录（直接改，无需同步）；
- “本地服务器” = 项目文件夹 ≠ 服务器文档根目录（需要上传/下载同步）；
- “远程服务器” = 服务器不在本机，靠网络协议传输文件。

### 二、前置条件：启用必要插件
所有和服务器连接（尤其是远程服务器）的功能，都依赖 `FTP/SFTP/WebDAV Connectivity` 插件，操作步骤：
1. 打开 GoLand 设置：`Cmd + ,`（Mac）/ `Ctrl + Alt + S`（Windows/Linux）；
2. 选择「插件」→「已安装」；
3. 找到 `FTP/SFTP/WebDAV Connectivity`，确保复选框勾选（默认已启用，功能不可用时检查这里）。

### 三、核心配置：服务器访问配置
无论哪种服务器类型，都需要定义“服务器访问配置”，它是 GoLand 和服务器交互的核心规则，包含这些关键信息：
1. **服务器类型**：就地/本地/远程（对应上面三种）；
2. **主机地址**：就地/本地服务器默认是本机，<span style="color: #ff6600; font-size: 16px; font-weight: bold;">远程服务器填远程主机 IP/域名</span>；
3. **配置根目录**：通过该配置<span style="color: #ff6600; font-size: 16px; font-weight: bold;">能访问的服务器最高级文件夹</span>（比如远程服务器的 `/var/www/html`）；
4. **访问 URL**：访问该根目录的网址（比如 `http://localhost:8080` 或 `http://192.168.1.100`）；
5. **传输协议/参数**：<span style="color: #ff6600; font-size: 16px; font-weight: bold;">远程服务器需填 SFTP/FTP 等协议的端口、账号密码</span>；本地/就地服务器无需；
6. **映射（mapping）**：<span style="color: #ff6600; font-size: 16px; font-weight: bold;">本地项目文件夹 ↔ 服务器目标文件夹 ↔ 访问 URL 的对应关系</span>（核心！比如本地 `./src` 对应服务器 `/var/www/html/src`，访问 URL 是 `http://localhost/src`）。

### 四、实用功能：默认服务器配置
为了简化操作，GoLand 允许将某一配置设为“默认”，后续无需手动选服务器就能自动执行操作，比如：
- 自动上传修改后的文件；
- 手动上传/下载文件时无需选目标；
- 对比本地和远程文件；
- 直接编辑远程单个文件。

#### 设置默认服务器的两种方式：
1. 方式 1（设置界面）：
    - 打开设置 `Cmd + ,` → 「构建、执行、部署 | 部署」；
    - 选中要设为默认的配置 → 点击工具栏「Use as default」按钮。
2. 方式 2（状态栏快捷操作）：
    - 点击 GoLand 底部状态栏的「默认部署服务器」→ 从菜单选目标服务器/服务器组。

#### 补充：配置默认服务器的上传规则
- 路径：设置 → 「构建、执行、部署 | 部署 | 选项」；
- 可配置：自动上传时机（比如保存文件时）、是否删除远程多余文件等。

### 五、进阶：服务器组
如果需要把代码部署到多个服务器（比如测试机+预生产机），无需逐个配置部署，直接创建「服务器组」，把多个服务器加入组，操作一次就能部署到所有服务器，避免重复工作。

---

### 总结
1. GoLand 的服务器配置分三类：就地（无同步）、本地（本地服务器+文件上传）、远程（网络协议传输），核心区别是“项目目录与服务器目录是否一致”“服务器是否在本机”；
2. 所有服务器交互都依赖「服务器访问配置」，其中「mapping（路径映射）」是本地文件和服务器文件关联的关键；
3. 设置「默认服务器配置」能简化上传/下载、远程编辑等操作，多服务器部署可创建「服务器组」提升效率。

这样梳理后，你可以根据自己的开发场景（本地开发/远程部署）选择对应的配置类型，核心先明确“项目文件和服务器文件的位置关系”，再配置映射和传输规则即可。
