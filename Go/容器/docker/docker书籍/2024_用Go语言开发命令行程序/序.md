## 一、译者序（杜万，2023年7月）

### 1. 为什么选择 Go 做命令行工具？
- **传统工具的痛点**：
    - 高级语言（如 Python/JavaScript）编写的命令行工具可移植性差，依赖运行时环境。
    - Bash 脚本虽然迭代快，但维护困难，且在不同环境（如 GNU vs BSD）下存在兼容性问题。
- **Go 的优势**：
    - <span style="color: #ff6600; font-size: 16px; font-weight: bold;">静态类型、编译型语言</span>，提供出色的性能和可靠性。
    - 语法简洁，标准库丰富，易于构建高质量的命令行工具。
    - <span style="color: #ff6600; font-size: 16px; font-weight: bold;">优秀的跨平台能力</span>，可轻松实现不同操作系统之间的可移植性。

### 2. 命令行工具的价值
- 在 AI 时代，命令行工具依然是现代软件开发的重要组成部分。
- 它在服务器管理、DevOps 流程、云计算和容器化环境中不可或缺，能极大提升自动化和效率。

### 3. 本书内容
- 学习如何使用 Go 构建各种命令行工具，包括交互式终端工具、REST API 客户端和字符图形工具。
- 涵盖文件处理、进程控制、数据库操作、分发工具等高级主题。
- 强调测试的重要性，每个示例都附带详细测试，保证交付品质。

## 二、前言（Steve Francia，Cobra、Viper、Hugo 作者）

### 1. 作者与 Go 的渊源
- 2012 年，作者为了学习 Go，决定创建一个静态网站生成器，这就是后来大名鼎鼎的 **Hugo**。
- 由于当时 Go 生态中没有合适的库支持“应用-命令-标志-参数”的设计模式，他自己编写了库，并将其剥离为独立项目：<span style="color: #ff6600; font-size: 16px; font-weight: bold;">**Cobra**（CLI 框架）和 **Viper**（配置管理）</span>。
- 这两个库后来成为了 <span style="color: #ff6600; font-size: 16px; font-weight: bold;">Go 生态中最核心的 CLI 工具库，被 Kubernetes、Docker 等项目广泛使用</span>。

### 2. Go 为什么适合命令行工具？
作者总结了 Go 的几大关键优势：
- **静态编译**：生成的可执行文件无需任何运行时依赖，部署和安装极其方便。
- **交叉编译**：可以在一台机器上为任何平台（Windows、macOS、Linux 等）构建二进制文件，消除了对构建机器的需求。
- **极快的构建速度**：开发体验流畅，感觉像在使用动态语言。
- **原生并发**：可以充分利用多核机器，提升程序性能。

### 3. 本书的定位
- 这不是一本简单的“食谱”，而是一本带你理解真正发生了什么的书。
- 它从基础概念开始，循序渐进，通过从简单工具到成熟命令行应用的旅程，让你重温学习 Go 时的兴奋感。
- 本书涵盖了构建 Hugo、Docker 和 Kubernetes 等应用程序所使用的 Go 技术和库，是一本“解锁新能力”的指南。
