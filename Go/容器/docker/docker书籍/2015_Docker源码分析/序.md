## 二、Docker 是什么

### 1. Docker 的官方定义
> “Build, Ship, Run. An open platform for distributed applications for developers and sysadmins.”
>
> （构建、分发、运行。一个面向开发者和系统管理员的分布式应用开放平台。）

### 2. Docker 解决的核心问题
- **开发与运维的鸿沟**：传统模式下，开发和运维职责分离，导致软件交付效率低、环境不一致。<span style="color: #ff6600; font-size: 16px; font-weight: bold;">Docker 提供了“一次构建，随处运行”的能力</span>，弥合了这一鸿沟。
- **轻量级虚拟化**：<span style="color: #ff6600; font-size: 16px; font-weight: bold;">从 Linux 内核出发，利用 Namespace 和 Cgroup 等技术，秒级提供应用隔离环境</span>，满足云计算时代的“隔离”需求。
- **镜像与容器技术**：
    - 镜像利用联合文件系统，将软件和运行环境打包，实现环境与应用的灵活结合。
    - Dockerfile 极大简化了镜像的构建和复用，为镜像的转移和重新构建提供了可能。

### 3. Docker 与云计算
Docker 提供了便捷的资源分配方式，解决了应用运行与系统环境的依赖问题，非常适合云时代的应用迁移和部署，因此被认为是“为云计算而生”的技术。

## 三、本书的内容结构

本书是一本引导读者了解 Docker 实现原理的技术书，内容主要分为三个部分：

1.  **第一部分：Docker 的架构**
    - 从宏观角度介绍 Docker 的整体架构设计和各模块的职责。

2.  **第二部分：Docker 的核心模块（主体部分）**
    - 深入分析 Docker 中多个重要模块，包括：Docker Client、Docker Daemon、Docker Server、Docker 网络、Docker 镜像、Docker 容器等。
    - 第2-8章：从架构入手，勾勒骨架；
    - 第9-11章：重点讨论 Docker 镜像技术；
    - 第12-14章：分析 Docker 容器的始末。

3.  **第三部分：Docker 生态三驾马车**
    - 介绍 Swarm、Machine、Compose 这三个工具，它们补充了 Docker 的跨主机能力和部署能力，是 Docker 生态中功能强大的软件。
