

这是《**Docker——容器与容器云**》（第2版）的推荐语、序、前言及本书结构说明，我帮你梳理一下核心信息：

---

### 一、这本书的定位与价值
- **国内原创深度解析 Docker 与 Kubernetes 的经典之作**：由浙江大学 SEL 实验室团队撰写，是国内较早深入剖析容器与容器云技术的原创图书。
- **从实践到原理的全面覆盖**：不仅讲解 Docker 与 Kubernetes 的使用方法，更深入源码层面（如 Docker 1.10、Kubernetes 1.2），剖析其 namespace、cgroups、网络、存储等核心原理。
- **行业认可度高**：得到了七牛云存储、数人科技、火币网、百度、华为、蚂蚁金服等多家企业技术负责人的推荐，被视为容器领域的重要参考书。

---

### 二、核心内容结构
本书分为两大部分，层层递进：

#### 第一部分：Docker 容器核心原理与实践
- **第1-2章**：快速上手 Docker，了解基本使用方法，为后续源码解析做铺垫。
- **第3章（核心）**：以 Docker 1.10 源码为基础，深入分析 namespace、cgroups 原理，从 `docker run` 命令出发，贯穿容器创建、镜像组织、联合文件系统及容器网络初始化的全流程，展示 Docker 源码的设计与执行逻辑。
- **第4章**：介绍“容器化思维”及 Docker 相关实战技巧（网络、监控、服务发现等），用平实语言解读代码背后的设计思想。

#### 第二部分：基于 Docker 的“容器云”平台
- **第5章**：介绍简单的容器云解决方案，对比 Docker 官方“三剑客”（Compose、Swarm、Machine）及经典 PaaS 项目（Flynn、Deis）。
- **第8章（核心）**：以 Kubernetes 1.2 源码为基础，从概念到架构，再到组件级源码解析，多维度讲解 Kubernetes 容器云平台的技术细节。
- 其他章节：更新了 Docker 命令行工具、“容器化思维”与“微服务”的讨论，完善了 Docker 安装指南及 Kubernetes 标准。

---

### 三、第2版的重要改进
- **版本升级**：Docker 从 1.6.2 更新到 1.10，Kubernetes 从 0.16 更新到 1.2，覆盖了生产就绪的关键版本。
- **Docker 架构更新**：
    - 解耦为运行时、镜像构建、镜像分发、网络、数据卷等独立功能组件。
    - 网络和数据卷提升为“一等公民”，具备独立生命周期。
    - 支持网络插件、存储插件，引入可信镜像分发机制。
    - 支持 runC 作为容器运行时，与 OCI 标准对齐。
- **Kubernetes 能力增强**：
    - 支持 ReplicaSet、Job、DaemonSet、Deployment、PetSet 等更多应用场景。
    - 兼容更多容器后端、网络及存储方案，增加认证与授权机制。
    - 讨论了社区新特性（如 StatefulSet、rescheduler）及与 OCI 的关系。
